1. **优先使用批量更新脚本**：比单条更新效率高10倍以上，23只股票批量更新仅需~20ms，查询列表仅需~25ms

### 详细联动说明（v2）

#### 1. 核心新增：持有数量
- **新增字段**：quantity（持有数量，股/份）
- **核心地位**：持有数量是整个计算的基础！
- **固定不变**：除非主动调整，否则持有数量不变
- **默认不展示**：列表默认不显示持有数量，可通过"显示数量"查看

#### 2. 持有数量 × 现价 → 个股市值
- **联动关系**：个股市值 = 持有数量 × 现价
- **触发条件**：添加股票时或更新现价时自动计算
- **示例**：持有100股股票A，现价100.00元 → 个股市值 = 100 × 100.00 = 30,000元

#### 3. 个股市值 ÷ 总资金 → 仓位%
- **联动关系**：仓位% = (个股市值 / 总资金) × 100
- **默认总资金**：由用户设置和定义，首次添加股票时可指定总资金参数
- **示例**：个股市值30,000元 → 仓位% = (30,000 / 600,000) × 100 = 10.0%

#### 4. 仓位% ↔ 模式
- **自动调整规则**：
  - 仓位 ≤ 2% → 自动调整为“分散”
  - 仓位 > 2% → 自动调整为“集中”
- **模式名称简化**：
  - "2%集中" → 显示为"集中"
  - "2%分散" → 显示为"分散"
- **示例**：仓位10.0% → 模式自动调整为"集中"

#### 5. 更新现价时全链路重新计算
- **联动触发条件**：更新现价
- **自动重新计算的内容**：
  1. 个股市值 = 持有数量 × 新现价
  2. 仓位% = (个股市值 / 总资金) × 100
  3. 模式 = 根据仓位自动调整
  4. 盈亏% = (新现价 - 成本价) / 成本价 × 100
  5. 盈亏金额 = 个股市值 × (盈亏% / 100)
- **固定不变**：持有数量、成本价、止损价

#### 6. 多条件综合 → 建议
- **建议由多个条件综合计算**：
  1. 先检查止损价相关规则
  2. 再检查模式和仓位
  3. 最后检查持有理由
- **固定不变**：建议不是数据库字段，每次列表时实时计算

# 股票风险控制策略（2%原则）

控制单笔交易最大风险为本金的2%，区分2%分散与2%集中，支持SQLite数据库记录多个股票信息。

## 核心概念

### 2%风险原则
任何一笔交易，最多只承担本金的2%风险。

### 两种模式
- **2%分散**：适合多品种分散投资
- **2%集中**：适合单品种趋势投资，计算仓位

### 数据表结构
**stocks表**：
- `id` - 主键
- `name` - 股票名称
- `code` - 股票代码
- `mode` - 模式（2%集中/2%分散）
- `position` - 持有仓位（%）
- `total_value` - 总值（基于总资金）
- `stop_loss` - 止损价
- `current_price` - 现价（每日更新）
- `pnl` - 盈亏金额
- `pnl_percent` - 盈亏百分比
- `created_at` - 创建时间
- `updated_at` - 更新时间
- `is_deleted` - 是否删除（0=否，1=是）
- `deleted_at` - 删除时间

**重要**：软删除机制，删除的数据只是对用户不可见，实际还保存在数据库中。

---

## 2%集中 仓位计算公式

### 公式
```
仓位 = 2% / (1 - 止损价 / 现价)
```

### 原理
- 止损跌幅 = (现价 - 止损价) / 现价 = 1 - 止损价/现价
- 当价格跌到止损价时，亏损 = 仓位 × 止损跌幅 = 2%

#### 批量更新股票现价（10个10个批量更新）
**原则**：一次性更新完所有股票，不要中途询问是否继续

**工具选择**：优先用DashScope WebSearch（通过mcporter），超时的话换Tavily

**超时时间**：mcporter默认超时10秒，可以用Python脚本设置60秒超时

**搜索时必须带股票代码**：
- 为了确保唯一性，搜索时必须带上股票代码
- 格式：`{日期} {股票名称} {股票代码} 股票现价`
- 例如：`22022年3月5日 股票A 000001 股票现价`

**批量更新方法（推荐）**：
- **10个10个批量更新**，效率更高！
- 先写一个Python脚本，一次性搜索10个股票
- 从搜索结果里提取价格，然后批量更新到数据库

**批量更新示例流程**：
1. 整理所有股票列表（名称 + 代码 + 数据库ID）
2. 写Python脚本，一次性搜索10个股票（间隔2秒）
3. 从搜索结果里提取价格
4. 批量更新数据库
5. 网络超时的话换个时间再试

**示例脚本（一次性搜索多个）**：
```python
import subprocess
import time

stocks = [
    ("股票A", "000001", 16),
    ("股票B", "000002", 17),
    ("科创50ETF华泰柏瑞", "588090", 18),
    # ... 更多股票
]

for name, code, stock_id in stocks:
    query = f"2222年3月5日 {name} {code} 股票现价"
    cmd = ["mcporter", "call", "dashscope-websearch.bailian_web_search", f"query:{query}", "count:3"]
    result = subprocess.run(cmd, capture_output=True, text=True, timeout=60)
    # 从result.stdout里提取价格，然后更新数据库
    time.sleep(2)
```

**美股换算人民币**：
- 汇率：6.9124（可根据需要更新）
- 例如：股票C 183.04美元 → 183.04 × 6.9124 ≈ 1265.22元


**数据库字段说明**：
- `updated_at`：每次更新现价时自动记录更新时间，用于知道哪些已更新哪些未更新

## 例子

### 例子1：上证50趋势交易
- **标的**：上证50
- **止损价**：2457（4月7日贸易战低点，本轮趋势起点）
- **现价**：2960
- **计划**：2%集中

**计算：**
- 止损跌幅 = 1 - 2457/2960 ≈ 17.06%
- 仓位 = 2% / 17.06% ≈ 11.7%

**结果：** 持有约11.7%仓位，在未打穿2457前，最多承担本金2%风险。

## 注意事项

- ✅ 止损价应该是关键支撑/阻力位，有技术面或基本面支撑
- ✅ 2%是单笔最大风险，不是目标亏损
- ✅ 到达止损位严格执行纪律
- ✅ 可以根据个人风险偏好调整（如1%、3%）
- ✅ 删除操作是软删除，数据还在数据库中
